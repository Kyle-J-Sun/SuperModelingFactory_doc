# v0.7.0

本次 minor release 将此前已 opt-in、并在 0.6.x 明确预告的治理默认值正式收紧，同时补齐 G06-2 分类变量 VIF 门。升级后，如需维持旧口径，请显式传入旧值；只有下文明确说明的参数仍以显式 `None` 保留 0.6.x 行为。

## 默认治理收紧

### OOT 使用边界

- `synthesize_missing_oot` 默认 `False`：没有真实 OOT 时，不再把 OOS 副本标成 OOT。CMP 不再创建 `"oot"` split；FVP 保留空 OOT 表示，相关下游步骤会跳过它。
- CMP 默认 `evaluation_splits=["ins", "oos"]`，默认模型评估、图表和 Excel 不再消费 OOT；`extra_eval_datasets` 仍是独立的 eval-only 输入。
- `search_eval_splits` 默认 `["oos"]`，LR search 与 Optuna 不再把 OOT 放进候选调参的 eval sets；`backward_validation_split` 继续是 `"oos"`，`backward_report_splits` 默认 `[]`，避免逐轮 backward 报告读取 OOT。
- 真实 OOT 仍可通过显式配置进入相应阶段；`forbidden_splits` 的消费点硬闸和 `split_governance` / 模型 metadata 审计字段继续生效。

### WOE 与评估方向

- FVP `woe_fit_scope` 默认从 `"all"` 变为 `"post_missing_gate"`：先执行 selection-grade 缺失率门，再对幸存变量拟合顶层 WOE，避免单个高缺失变量使整组 WOE 产物失效。需要历史口径时显式传 `"all"`。
- `MonotoneWOEBinner.refine_min_n_bins_policy` 默认从 `None` 变为 `"warn"`：`refine_dtree()` 若把普通箱数压到 `min_n_bins` 以下会留下可见告警；`"enforce"` / `"raise"` 仍可按更严格的治理需求使用，显式 `None` 可关闭该检查。
- `gains_ascending` 默认 `True`：分数升序、bin 1 为低分/低风险。Gains summary、加权 Gains 和评估分布图使用同一方向，消除了旧版表格与图表路径的方向漂移；显式 `False` 统一降序，显式 `None` 保留 0.6.x 各路径历史方向。

## G06-2：分类变量 VIF 门

`selection_params` 新增 `vif_use_woe_bins`，默认 `False`，并保留原始数值 VIF 的兼容语义：

- Raw VIF 只把数值幸存变量送入 statsmodels；字符串、object 与其他非数值列会保留在选择结果中，不会因 VIF 被误删。运行会发出 `UserWarning`，并在 selection summary / VIF stage artifact 中持久记录排除列；数值不足时以 `skipped_insufficient_numeric` 跳过该门。
- bool 和 pandas nullable numeric/boolean 只在 VIF 矩阵中转成浮点，普通 NumPy 数值输入保持原有行为。
- `vif_use_woe_bins=True` 时，VIF 基于 INS 的 WOE 编码矩阵，分类变量可参与共线性判断。该路径惰性创建 adapter、尊重自定义 WOE 后缀、在副本上 transform，并会对预拟合 engine 覆盖不完整给出可操作的错误。
- 对声明的分类变量，WOE-VIF 需要 `woe_engine="monotone"`；`equal_freq` 的 `WOE_Master` 不支持这一分类拟合路径，会在拟合前明确报错。

FVP 已把该字段透传到筛选配置，并在 `selection_config_snapshot` 中记录最终值。

## 迁移提示

若业务确实需要恢复旧行为，请显式设置相应值，例如：

```python
CreditModelPipelineConfig(
    synthesize_missing_oot=True,
    evaluation_splits=["ins", "oos", "oot"],  # 仅在真实 OOT 存在时
    search_eval_splits=["oos", "oot"],
    backward_report_splits=["oot"],
    gains_ascending=None,  # 精确保留 0.6.x 各评估路径的历史方向
)

FeatureValidationPipelineConfig(
    woe_fit_scope="all",
    monotone_woe_params={"refine_min_n_bins_policy": None},
)
```

显式消费 OOT 前，应先确认它是业务真实 OOT，而不是历史版本合成的 OOS 副本。
